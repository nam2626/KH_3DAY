객체지향프로그래밍(OOP)
	현실세게에 있는 사물이나 대상을 그대로 모델링해서 코드로 반영

클래스
	속성(State) : 객체의 특징이나 상태를 나타내는 데이터(학생 이름, 자동차 모델명...)
	행동(Behavior) : 객체가 할수 있는 동작이나 기능(학생이 자기소개하는것, 자동차가 가속하는 것)
	
	코드에서는 속성 : 변수(field) 행동 : 메서드(method)
	
상속(Inheritance)
		- 부모 클래스가 자식 클래스에게 가지고 있는 필드 및 메서드를 물려주는 행위
		- 자식 클래스는 부모 클래스가 아니기 때문에, 
		  부모클래스의 private 멤버에 접근할 수 없음
		- 생성자는 상속의 대상이 아님
		- 자바는 하나의 부모클래스만 가질 수 있다.
	
	부모 클래스 - 자식 클래스
	슈퍼 클래스 - 서브 클래스
	상위 클래스 - 하위 클래스
	
	상속 장점
		- 코드를 재사용 할 수 있음.
		- 부모와 자식 관계를 만들어 줌 --> 자식 클래스가 부모 클래스로 형변환이 서로 가능
	
	오버라이딩(Overriding) : 부모로부터 상속 받은 메서드를 재정의
						 ※메서드명 동일, 매개변수도 동일, 접근제어자도 동일
						 
	오버로딩(Overloading) : 메서드 중복 정의 방법
						  1. 매개변수 개수가 다르거나
						  2. 매개변수 개수가 같다면, 매개변수 타입을 서로 다르게 하면 됨
